# deep-research

Conducts exhaustive, multi-layered research using two rounds of parallel investigation with a tiered model strategy. Round 1 spawns broad-sweep Sonnet agents across 5-7 themes. An Opus gap-analysis identifies weak spots, then Round 2 launches targeted collectors plus a dedicated Opus skeptic agent that challenges majority findings. Produces comprehensive cited reports with per-finding confidence levels, contradiction analysis, and source credibility assessment.

Use when the user needs research that goes beyond a quick overview — when depth, rigor, and explicit uncertainty matter.

## Usage

```
/deep-research <topic or question>
```

- `/deep-research What are the security implications of WebAssembly in production?`
- `/deep-research How do distributed consensus algorithms compare for financial systems?`

## What It Does

The skill runs a 7-phase research pipeline over ~10-12 minutes:

1. **Scope & Plan** — Decomposes the topic into 5-7 themes and presents a research plan for user review
2. **Broad Sweep (Round 1)** — Spawns 5-7 Sonnet agents in parallel for fast data collection across all themes
3. **Gap Analysis** — An Opus agent analyzes Round 1 findings to identify gaps, contradictions, and weak claims
4. **Targeted Deep Dives (Round 2)** — 3-4 Sonnet collectors target specific gaps, plus 1 Opus skeptic agent challenges the strongest claims
5. **Cross-Reference & Synthesize** — An Opus agent merges all findings, cross-validates across sources, and assigns per-finding confidence levels
6. **Report Generation** — Produces a comprehensive report with executive summary, 12-20 key findings, deep dives, contradictions, and source assessment
7. **Self-Check** — Audits every citation, flags single-source claims, and verifies contradiction completeness

## Research Insights

This skill's design was informed by extensive research into how leading deep research systems work:

- **Two-round architecture** is used by all leading systems (OpenAI, Gemini, Perplexity) — broad sweep followed by targeted deep dives based on gap analysis
- **Citation failure rates exceed 60%** in AI search (Tow Center/Columbia study) — this skill mitigates with strict citation-audit rules and single-source flagging
- **"Chat-Chamber effect"** causes AI to confirm rather than challenge assumptions — addressed with a dedicated Opus skeptic agent
- **Heterogeneous agents** (different model tiers) achieve ~91% vs ~82% accuracy — this skill uses Sonnet for collection and Opus for analysis
- **58% of users want explicit confidence scores** but none of the major tools provide them — this skill tags every finding with HIGH/MEDIUM/LOW confidence

## Design Decisions

- **Structure**: Standard (SKILL.md + references/) — needs credibility framework and report template as reference files
- **Pattern**: Phase-based workflow (7 phases) — multi-round research requires clear sequential phases with parallel agents within each phase
- **Model strategy**: Tiered — Sonnet for data collection (fast, parallel), Opus for analysis/synthesis/skeptic (highest quality reasoning)
- **Skeptic pattern**: Dedicated Opus agent in Round 2 that challenges majority findings regardless of confidence — addresses "minority correction asymmetry" from multi-agent debate research
